OLTP Database Systems

16 Credits · SaaS Tiers, Self-Hosted mit RBS
Misch ▼
d) OLTP SaaS — Standard vs IO-Optimized
Rechnung
Standard: Storage + Compute + IO = $3,072 + $360 + $2.40 = $3,434
IO-Opt: $4,608 + $540 + $0 = $5,148 → Standard gewinnt
Exakt
Exercise 12 — Aurora Tab: Identisches Schema — Standard (Storage + Compute + $/IO) vs IO-Optimized (höherer Storage/Compute, keine IO-Kosten). In Ex12 gewann IO-Opt bei 52K IOPS, hier gewinnt Standard bei nur 400 IOPS/Tag.
Ex12: Aurora Standard $27,807 vs IO-Opt $1,202 (52K IOPS → IO-Opt gewinnt) Klausur: Standard $3,434 vs IO-Opt $5,148 (400 IOPS → Standard gewinnt) KEY INSIGHT: Bei wenigen IOPS → Standard; bei vielen IOPS → IO-Optimized!
e) Self-Hosted: Instance + Remote Block Storage
Rechnung
RBS: 20 TB × $0.10/GB + 400 IOPS × $0.36 = $2,048 + $144. Instance: 8 vCPU → r8.2xlarge ($0.3125/h × 720 = $2,880). Total: $5,072
Exakt
Exercise 12 — EBS Tab: Identische Struktur. gp3/io2 Storage + provisioned IOPS + Instance. Hier mit TUMCloud Pricing Sheet statt AWS, aber die Methodik ist 1:1.
Ex12: gp3 = $338 EBS + $525 instance = $863 total Klausur: RBS = $2,192 + r8.2xlarge $2,880 = $5,072

Storage Application (Video Campaign)

12 Credits · Object Store, Durability, Network
Rechnung ▼
a–c) Storage, PUT, GET Kosten (Standard + Express)
Rechnung
Standard: Storage $625 + PUT $1,000 + GET $40 = $1,665
Express: Storage $3,125 + PUT $8,312.50 + GET $118.13 = $11,555.63
Exakt
Exercise 4 (Object Store Tabs), Exercise 7, Exercise 11 (S3 vs S3 Express) — Alle drei üben exakt dieses Muster: Storage-Kosten (GB × Preis) + Request-Kosten (Anzahl/1000 × Preis) + bei Express: Data Transfer (GB × Preis). Wurde in 3 Übungen wiederholt!
Formeln identisch in allen Übungen: Storage = size_GB × $/GB/month Requests = count / 1000 × $/1000req Express Data = size_GB × $/GB (zusätzlich!)
d) Durability — Wahrscheinlichkeit Objektverlust
Rechnung
p = 10⁻⁶ pro Objekt. P(≥1 Verlust) = 1 − (1 − 10⁻⁶)^200,000 = 18.13%
Exakt
Exercise 11 — Durability Tab: Exakt die gleiche Formel P(loss) = 1 − (1−p)^n. In Ex11 mit AFR und HDDs, hier mit Objekt-Durability. Selbe Mathematik, anderer Kontext.
Ex11: P(loss,1yr) = 1 − (1 − 0.003504)^142 = 39.3% Klausur: P(loss) = 1 − (1 − 10⁻⁶)^200000 = 18.13%
e–f) Network Kosten (Same Region + Cross-Region Fehler)
Rechnung
e) Nur VM→Internet: 100K × 16MB / 1024 × $0.50 = $781.25
f) + Cross-Region: zusätzlich OUT + IN Region = $1,718.75
Konzept
Kein direktes Übungsäquivalent mit Netzwerkkosten-Berechnung. Das Pricing Sheet (Network Charges) muss man kennen. Exercise 12 (HADR) behandelt Inter-AZ Kosten, aber inter-Region + Internet Egress war neu in der Klausur.
